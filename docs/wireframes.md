## Логика и иерархия контента

### 1. Hero-секция
**Цель:** Немедленное понимание ценности и призыв к действию

**Элементы:**
- Геометрическая анимация (слева) - абстрактные формы, вращающиеся элементы
- Заголовок с ценностным предложением (крупный, 64px) - справа
- Подзаголовок с уточнением (20px) - справа
- Primary CTA кнопка (яркая, градиентная) - справа
- Secondary CTA ссылка (менее навязчивая) - справа
- Статистика (социальное доказательство) - справа

**Иерархия:**
1. Заголовок (самый важный)
2. Подзаголовок (контекст)
3. CTA кнопки (действие)
4. Статистика (доверие)

### 2. Решения/Преимущества
**Цель:** Быстрое понимание ключевых преимуществ продукта

**Элементы:**
- Заголовок секции
- Подзаголовок
- Сетка из 6 карточек (3 колонки на desktop, 2 на tablet, 1 на mobile)
- Каждая карточка: иконка, заголовок, текст

**Иерархия:**
1. Заголовок секции
2. Карточки с преимуществами (равная важность)
3. Визуальная иконка → заголовок → описание

### 3. Как это работает
**Цель:** Снижение барьера входа, показ простоты процесса

**Элементы:**
- Заголовок секции
- Подзаголовок
- 3 шага в ряд (горизонтальная последовательность)
- Каждый шаг: номер, заголовок, описание
- Визуальные стрелки между шагами (на desktop)

**Иерархия:**
1. Номер шага (визуальное выделение)
2. Заголовок шага
3. Описание шага

### 4. Отзывы/Кейсы
**Цель:** Социальное доказательство перед конверсией

**Элементы:**
- Заголовок секции
- Подзаголовок
- 3 карточки с отзывами (3 колонки на desktop, 1 на mobile)
- Каждая карточка: логотип компании, рейтинг, текст отзыва, метрики, автор

**Иерархия:**
1. Рейтинг и логотип (первое впечатление)
2. Текст отзыва (основное сообщение)
3. Метрики (конкретные результаты)
4. Автор (доверие)

### 5. CTA-форма
**Цель:** Конверсия в лид

**Элементы:**
- Двухколоночная композиция
- Левая колонка: заголовок, подзаголовок, список преимуществ
- Правая колонка: форма с полями (имя, компания, email, телефон, сообщение), CTA кнопка

**Иерархия:**
1. Заголовок (мотивация)
2. Форма (действие)
3. Преимущества (убеждение)

## Адаптивность

### Desktop (1440px)
- Полная ширина контейнера (1280px max)
- Все колонки видны (3 колонки для решений и кейсов)
- Горизонтальная композиция для "Как это работает"
- Двухколоночная форма

### Tablet (768px)
- Уменьшенные отступы
- 2 колонки для решений
- Вертикальная композиция для шагов
- 1-2 колонки для кейсов
- Вертикальная форма (стек элементов)

### Mobile (375px)
- Одноколоночная структура везде
- Уменьшенные размеры шрифтов
- Полная ширина кнопок
- Стек всех элементов
- Оптимизация для touch-интерфейса

## Визуальный поток пользователя

**Z-паттерн сканирования:**
1. Hero (верхний левый угол - внимание)
2. Решения (поперек, затем вниз)
3. Как это работает (поперек)
4. Кейсы (поперек)
5. Форма (вниз, действие)

**F-паттерн для текстового контента:**
- Заголовки слева (сканирование)
- Основной текст по центру/слева
- CTA справа или по центру (финальное действие)

## Ключевые дизайн-решения

1. **Белое пространство:** Много воздуха между секциями для фокуса
2. **Визуальная иерархия:** Размер шрифта и цвет создают четкую структуру
3. **Контраст:** Яркие CTA на фоне нейтральных секций
4. **Повторение:** Консистентные карточки и кнопки
5. **Выравнивание:** Все элементы выровнены по сетке для порядка
